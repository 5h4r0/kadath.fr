## Tâches (dans l'ordre)

### 1. [BUG-1] DOMPurify manquant avant stockage TipTap — CRITIQUE (XSS)

**Fichier :** `src/app/actions/cms.ts`, fonction `updatePage`

Actuellement, `sections` est stocké en DB sans sanitisation. Or `src/app/[locale]/(public)/[slug]/page.tsx` rend le contenu via `dangerouslySetInnerHTML`. Risque XSS.

**Fix :**
```ts
import DOMPurify from 'isomorphic-dompurify'
```

Dans `updatePage`, si `data.sections` est présent, le convertir en HTML via `tiptapToHtml`, sanitiser, puis re-sérialiser avant le `.update()`.

Ou plus simple : `TipTapEditor` appelle `updatePage` avec `e.getJSON()` (objet TipTap JSON, pas du HTML brut). Dans ce cas, la protection est côté lecture — appliquer DOMPurify dans `tiptapToHtml.ts` sur le résultat HTML avant de le retourner. Choisir l'approche la plus sûre et la documenter.
